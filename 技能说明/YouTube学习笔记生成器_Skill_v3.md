# YouTube 学习笔记生成器

将 YouTube/B站 授课或访谈视频转化为一份**完整的学习文档**：读者读完等于认真听了一遍课，还带走了复习用的概念速查和可执行的行动指南。信息全覆盖，但读起来像有人在给你讲，而不是机器在提取数据。

## 何时使用

**自动触发条件**（符合任一即触发）：
- 用户提供 YouTube/B站 链接 + 以下任一关键词：笔记、整理、总结、学习笔记、学习文档、听课、做笔记
- 用户说："帮我整理这个视频"、"把这个视频内容总结一下"、"生成学习笔记"、"生成学习文档"
- 用户说："帮我听课" + 视频链接
- 用户输入 `/` 后手动选择本技能

**不触发的情况**：
- 仅提供链接，没有笔记/整理相关意图
- 要求翻译、提取字幕、下载视频（这些用其他工具）

## 输入输出

- **输入**：YouTube/B站 视频链接（+ 可选的格式偏好）
- **输出**（最终只保留 HTML 学习文档 + 字幕文件）：
  1. **HTML 学习文档**（唯一交付格式）：带排版设计、可视化图表的完整学习文档
  2. **字幕文件**（保存备用）：原始字幕文本（TXT 或 JSON 格式）
- **输出位置**：当前工作目录（HTML 文档以独立文件夹形式存放），最终打包为 ZIP 交付到桌面
- **不单独输出 Markdown 和 PDF**——用户可通过浏览器"打印为 PDF"自行导出

---

## 核心原则

> **不是做摘要，而是做"学习替身"——覆盖全部重要信息，但用自然叙述呈现。**
>
> **写完自己读一遍：如果读起来像"机器提取的数据报告"或"百度百科词条"，就重写。**
>
> **内容少了的笔记=废笔记。宁可多写不可少写。**
>
> **过程还原不到位=转述。如果笔记只给结论不给过程，就是高级摘要。**
>
> **过程还原≠逐句翻译。提取逻辑骨架，去除口语填充。宁可信息密度高，不要水分多。**
>
> **推导步骤丢失=废笔记。读者知道"结论是什么"，但不知道"为什么"。**
>
> **最好的学习文档不只是记录了什么，还帮读者记住了什么、能用什么。**

## 第一步：判断视频类型

拿到链接后，先根据标题和简介判断视频类型，不同类型采用不同的结构侧重：

| 视频类型 | 识别特征 | 结构侧重 | 典型例子 |
|---------|---------|---------|---------|
| **授课/讲座类** | 课程、讲座、分享、演讲 | 按知识点递进，强调逻辑链条和论证路径 | 大学课程、TED、技术分享 |
| **访谈/对话类** | 对话、访谈、播客、对谈 | 按话题切分，保留双方互动，突出核心嘉宾观点 | The Diary of a CEO、播客节目 |
| **教程/操作类** | 教程、教学、上手、从零开始 | 按步骤切分，强调可复现性，附代码/命令/截图说明 | 编程教程、软件操作、手工教程 |
| **故事/案例类** | 故事、传记、案例、复盘 | 按情节推进，保留叙事节奏，结尾提炼启示 | 人物传记、商业案例分析 |

> 如果类型不明确，默认按「访谈/对话类」处理。

## 第二步：获取视频内容（多级降级）

优先获取最完整的信息源，失败则逐级降级，确保任何环境下都能产出有价值的内容。

### Level 1：完整字幕（最佳）
1. 使用 `scripts/fetch_transcript.py` 获取完整字幕
   - **自动识别平台**：YouTube / Bilibili 链接自动识别
   - **YouTube**：优先用 `youtube-transcript-api`，失败自动降级 `yt-dlp`
   - **Bilibili**：自动用 `yt-dlp` 提取字幕
   - **跨平台**：Windows / macOS 均可用
   - **交付桌面**：`--to-desktop` 参数自动检测操作系统桌面路径
   - **重试机制**：`--retry N` 设置网络重试次数（默认 3 次）
   - **yt-dlp优先**：`--ytdlp-first` 强制优先使用 yt-dlp

   用法参考：
   ```bash
   # YouTube（输出到当前目录）
   python scripts/fetch_transcript.py https://www.youtube.com/watch?v=xxxxx

   # YouTube（直接交付桌面，AI 自动打包）
   python scripts/fetch_transcript.py https://youtu.be/xxxxx --to-desktop

   # Bilibili
   python scripts/fetch_transcript.py https://www.bilibili.com/video/BVxxxxxxxxxx

   # 指定语言 + 重试
   python scripts/fetch_transcript.py <URL> --lang zh-Hans --retry 5
   ```
2. 获取后立即向用户报告：
   - 视频标题
   - 总时长
   - 视频类型判断
   - 预估主题数（按每 15-20 分钟一个主题估算）
   - 快速扫描到的人名/数据/案例清单

### Level 2：第三方摘要平台（次优）
字幕获取失败时，尝试以下平台获取要点：
- VideoHighlight（videohighlight.com）
- Glasp（glasp.co）
- Eightify（eightify.app）
- 其他 AI 视频摘要工具

> 注意：摘要平台的内容是二手信息，必须标注"基于第三方摘要整理，可能有遗漏"。

### Level 3：网络文字回顾（可用）
如果摘要平台也不可用，通过搜索引擎查找：
- 博主的文字版回顾/笔记
- 论坛讨论（Reddit、知乎、V2EX 等）
- 相关书籍/文章的引用
- 嘉宾的其他公开演讲/文章

> 基于 Level 3 产出的笔记，必须在开头明确说明"字幕不可获取，基于公开资料整理"。

### Level 4：框架性笔记（保底）
以上都失败时，基于视频标题、简介、评论等有限信息，生成一份：
- 该领域核心概念的框架性梳理
- 嘉宾的主要思想和代表观点
- 延伸学习资源清单
- 明确标注"无法获取完整内容，以下为相关主题拓展"

> 绝不因为拿不到字幕就告诉用户"做不了"。

## 第三步：强制预扫描

通读可获取的全部内容，建立「内容清单」（内部流程，不输出给用户）：
- 总主题数、总子论点数
- 人名清单（全部，含身份背景）
- 书名/文章名/论文名（全部）
- 案例/故事清单（全部，标注时间范围）
- 关键数据/统计（全部）
- 关键引语（每个主题至少 3 句备选）
- 反问/讽刺/夸张/戏剧化段落（全部）
- 提到的工具/方法/实践建议（全部）

> 写完笔记后必须拿清单逐条核对，漏一项就返工。

---

## 第四步：主题切分

按**内容逻辑**切分主题，每个主题对应一个完整的"教学单元"：
- 每个主题包含 **4-6 个子论点**，不得少于 4 个
- 如果某个主题信息量不足 4 个子论点，合并到相邻主题
- 不得为了凑数把一个主题拆成多个薄弱主题
- 访谈类视频：按话题自然转换切分，不要按时间均分
- 教程类视频：按操作步骤/模块切分，确保每个主题可独立复现

## 第五步：撰写学习文档

### 文档整体结构

```
┌─────────────────────────────────────┐
│  封面区（标题、嘉宾、时长、简介）     │
├─────────────────────────────────────┤
│  为什么值得看（100字价值定位）       │
├─────────────────────────────────────┤
│  目录导航                           │
├─────────────────────────────────────┤
│  整课骨架（论证路径概括）            │
├─────────────────────────────────────┤
│  全局概念导图（Mermaid）            │
├─────────────────────────────────────┤
│  主题一 ~ 主题N（核心内容）          │
│  每个主题包含：                      │
│  · 时间段标注                       │
│  · 4-6个子论点（过程还原体）         │
│    · 结论在推导过程末尾自然浮现       │
│    · 还原讲师从A→B→C→D的思考路径    │
│  · 穿插引语、表格、图表              │
│  · 主题小结/概念导图                │
├─────────────────────────────────────┤
│  核心概念速查表                      │
├─────────────────────────────────────┤
│  行动指南 / 实践建议                 │
├─────────────────────────────────────┤
│  延伸阅读与资源                      │
├─────────────────────────────────────┤
│  参考来源                            │
└─────────────────────────────────────┘
```

### 每个主题的撰写要求

对每个主题写出**过程还原式叙述**：
- 用编号子论点组织
- 关键引语穿插在叙述中
- **结论在子论点末尾自然浮现，不在开头暴露**
- 人物首次出现：中文名 + 英文名 + 一句话身份简介
  - 例：丽莎·费尔德曼·巴雷特（Lisa Feldman Barrett），东北大学心理学教授，情绪构建理论提出者

**核心原则：先过程，后结论，高密度**

每个子论点必须还原讲师的**推导过程**，而不是只记录最终结论：

1. **起点（问题/现象/困惑）**：讲师从什么观察或问题出发？让读者先进入讲师的思考起点，而不是先知道答案
2. **推导过程（步骤还原）**：还原讲师的每一步推理。如果讲师用了3个论证步骤到达结论，笔记必须写3个步骤，每个步骤不少于30字。**禁止把3步压缩成1步**（"总之他认为..."）。**也禁止把1步硬扩成3步**（添加无意义的过渡和重复）
3. **关键转折/回应**：如果讲师在过程中遇到了反对意见、质疑、困惑，然后逐步回应，必须保留质疑→回应的完整来回。不能只写最终回应
4. **结论自然浮现**：结论在子论点末尾自然出现，不是开头就暴露。读者读完这段，像跟着讲师走完了一段思考旅程，最后到达终点
5. **衔接到下一段**：与下一个子论点的逻辑关联

> **字数是下限，不是目标。达标即可，不要硬凑。** 一个步骤如果20字就能说清楚逻辑，不要硬扩到50字。信息密度 > 字数长度。
>
> **避免口语化逐句还原**：不要写"他说…然后他说…接着他又说…"这种翻译腔。提取逻辑骨架，用因果关系连接，而不是按时间顺序复述每一句话。

> 注意：字数按**有价值的推导步骤**计算，不是按字幕原文的总字数计算。如果讲师在推导过程中有琐碎停顿、重复表述、无关过渡，这些不计入"步骤数量"。只保留有思想价值的推导步骤。

**过程还原的分层策略**

不是所有过程都需要完整还原。根据内容的**思想价值密度**决定还原深度：

| 内容类型 | 过程还原策略 | 理由 |
|---------|-------------|------|
| **核心观点/关键转折** | **完整还原**（保留犹豫→补充→修正的完整过程） | 这里展示了思维的复杂性、概念的重新定义、反直觉洞察，本身就是内容价值所在 |
| **次要观点/辅助说明** | **保留关键转折点**，压缩琐碎细节 | 保留转折，删除无价值的停顿和语言组织 |
| **事实性回答** | **直接陈述结论** | 事实性内容没有思想演进过程 |
| **寒暄/过渡** | **融合到上下文，一句话带过** | 不值得展开 |

**判断标准**：
- 这个过程是否展示了**思维的复杂性**？（犹豫是因为概念很难界定）
- 这个过程是否揭示了**反直觉洞察**？（最初的回答是不准确的，需要修正）
- 这个过程是否体现了**内心挣扎**？（回答这个问题对嘉宾来说有情感分量）
- 如果以上都不是 → 这个过程不值得完整还原

**过程完整性检测**：
写完每个子论点后，必须做以下检查：
- 讲师在这个话题上用了几步推导？笔记保留了相同数量的步骤吗？
- 如果讲师用了5步，笔记只写了1步 → **返工**
- 如果子论点开头50字以内就出现了结论性表述（"X认为Y是Z""X的核心观点是""X证明了"）→ **返工**，把结论移到段落末尾
- 如果讲师先质疑再回应，笔记只保留了回应结论 → **返工**

**访谈类的特殊要求——对话还原**：
- 如果嘉宾先犹豫/部分回答/补充/修正，笔记只保留了最终答案 → **返工**，必须保留犹豫→补充→修正的完整过程
- 如果嘉宾说"有人可能会反对..."然后逐步回应，笔记只记录了回应结论 → **返工**，必须保留质疑和回应的来回

## 第六步：学习增强模块

在主体内容之后，加入以下增强模块（根据视频类型选择适用的）：

### 1. 核心概念速查表（必选）
| 概念 | 一句话定义 | 实践意义 |
|------|----------|---------|
| xxx | ... | ... |

> 从全文中提炼 8-15 个最核心的概念，每个概念不超过 30 字定义 + 20 字实践意义。方便复习时快速回顾。

### 2. 行动指南 / 实践建议（必选）
将视频中所有可操作的建议、方法、练习汇总整理，按可执行性排序：
- **立即可做**（今天就能开始的小练习）
- **短期养成**（1-4 周的习惯养成）
- **长期修炼**（需要持续投入的方向）

> 让读者看完不仅"知道了"，还"知道怎么做"。

### 3. 自测思考题（可选，适合授课/教程类）
每章 2-3 道思考题：
- 理解类：检验是否掌握核心概念
- 应用类：要求读者结合自身经验思考
- 拓展类：引导进一步探索

### 4. 延伸阅读与资源（必选）
整理视频中提到的：
- 书籍（书名 + 作者 + 一句话推荐理由）
- 论文/文章（标题 + 来源）
- 人物（名字 + 代表作/核心思想）
- 工具/网站/课程
- 相关视频/播客

### 5. 金句卡片（可选，适合访谈/演讲类）
精选 3-5 句最有冲击力的原话，做成突出引用区块。

## 第七步：可视化规范

根据内容类型选择最合适的可视化形式，不要为了加图而加图。

### 可视化类型选择

| 内容类型 | 推荐可视化 | 工具 |
|---------|-----------|------|
| 概念层级/分类 | 思维导图 | Mermaid mindmap |
| 流程/机制/步骤 | 流程图 | Mermaid flowchart |
> 注意：Markdown table 指用 Markdown 语法编写的表格，不是输出 Markdown 文件。
| 数据/统计 | 柱状图 / 饼图 / 折线图 | ECharts |
| 时间序列 | 时间线 | HTML / Mermaid gantt |
| 核心概念/关键数据 | 信息卡片 | HTML cards |
| 因果/关系网络 | 关系图 | Mermaid graph |

### 基本要求
- 每个主题至少包含 **1 个可视化元素**（表格、图表、导图、卡片任选其一）
- 全文至少包含 **1 个全局概念导图**
- 数据类内容必须配图表，不能只用文字描述数字
- 每一张图/表都要有标题或说明
- 避免同一种可视化形式重复使用超过 3 次——混合使用更有节奏感

## 第八步：生成文件

### 输出优先级
1. **HTML 学习文档**（唯一输出）：直接编写或转换，包含完整排版和可视化
   - 输出为独立文件夹：`{视频标题}_学习文档/`
   - 包含：主 HTML 文件 + 内嵌 CSS（零外部依赖）

### 打包与放置到桌面

生成完所有文件后，统一打包为一个压缩包并放到用户桌面，方便用户找到和分享。

#### 打包规则
- **打包格式**：ZIP 压缩包
- **压缩包命名**：`{视频标题}_学习笔记.zip`（使用中文标题，用户一眼能认出来）
- **压缩包内容**：
  - HTML 学习文档文件夹（含主 HTML 文件 + 嵌入 CSS 的样式，零外部依赖）
  - 字幕文件（原始字幕 TXT/JSON，备用参考）

#### 放置位置
- **桌面路径**（按操作系统自动判断）：
  - macOS：`~/Desktop/`
  - Windows：`C:\Users\{用户名}\Desktop\`
  - Linux：`~/Desktop/`
- 如果无法确定桌面路径 → 放到当前工作目录，并告知用户

#### 打包命令参考

```bash
# macOS / Linux
zip -r "~/Desktop/{视频标题}_学习笔记.zip" "{视频标题}_学习文档/" "{视频标题}_字幕.txt"

# 或用 tar
tar -czf "~/Desktop/{视频标题}_学习笔记.tar.gz" "{视频标题}_学习文档/" "{视频标题}_字幕.txt"
```

### 自动清理（任务完成后执行）

**打包交付完成后，立即删除工作目录中所有中间产出**，只保留最终交付物。这是强制步骤，不是可选。

**必须删除的中间文件**：
- 主题切分文件（T1, T2… 或 topic_1, topic_2…）
- 子代理生成的分块文件（themes_1_2_3.md, notes_part1.md 等）
- 字幕原始文本（已保留 JSON 在压缩包中，工作目录不再保留 TXT 冗余）
- 所有临时脚本、测试文件、PDF 中间文件
- 之前任务遗留的旧笔记文件（与当前视频无关）

**允许保留的文件**：
- 最终 HTML 学习文档（在桌面文件夹中）
- 字幕 JSON 文件（如用户需要二次处理）
- Skill 文件本身

**工作目录最终状态**：打包完成后，工作目录应该只含当前视频的字幕 JSON 和当前 HTML 文件（如果桌面操作失败时的备用）。其余全部清理。

### 交付给用户

## 叙述规则（必须遵守）

### 1. 叙述体：像跟着听课，不要像读完摘要

**核心问题**：笔记最常见的质量失败不是"信息遗漏"，而是"过程压缩"——只保留了讲师的最终结论和一两个例子，丢失了从A到B到C到D的推导过程。读者知道"结论是什么"，但不知道"为什么"。

**禁止机械罗列**：
```
❌ 坏：- 讲师解释... - 讲师提到... - 讲师指出...
```

**禁止结论先行的转述**：
```
❌ 坏写法（结论先行，过程压缩为3句话）：
1. **Greene 认为权力是内在控制感**：他说权力不是政治地位，而是控制
自己的能力。他引用了Malcolm X的话。他还说外表很重要。他说人类是
灵长类动物。他举了自己被解雇的例子。
（这段在开头就给了结论，然后用并列句堆砌论据。读者看到的是"答案+要点"，
丢失了推导过程。这就是转述。）
```

**必须写成过程还原**：
```
✅ 好写法（先过程后结论，还原推导步骤）：
1. **Greene 先打破一个根深蒂固的误解**：大多数人一想到权力，脑海中
浮现的是政治人物、暴君或企业巨头。但 Greene 邀请我们换一副眼镜——
他问：当你面对孩子、配偶或同事时，你是否感到自己的声音被听见？这种
"能够施加影响"的感觉，才是权力的真谛。反过来，那种一切滑出掌控、
你的意见无人理会的处境，Greene 称之为"人类能经历的最悲惨的感受"。

   然后他用 Malcolm X 做了一个反常识的推进...接着他用自己被解雇的
惨痛经历让这个论点落地...最后他把一切收束到"自我控制"这个核心。

   > "If you can't control yourself, then you're in a lot of trouble..." 
   > ——Robert Greene

   （结论在段落末尾自然浮现，读者跟着走完了推导路径。）
```

**必须遵守的规则**：
- 用编号（1. 2. 3.）组织子论点
- 每个子论点开头有**粗体引导语**，但引导语不能预告结论（如"Greene 认为..."），应从问题/现象出发（如"Greene 先打破一个误解..."）
- 引语穿插在叙述中，用 `>` 引号标出
- 用"首先""然后""接着""这里值得注意的是""讲师转而指出""最后他总结道"等连接词还原时间线
- 低价值过渡融入上下文，不单独成条
- 读起来像"有人带你一步一步走过了讲者的思考过程"，而不是"有人告诉你讲者最后说了什么"

### 2. 信息完整：人名、书名、数据、案例全部保留

必须保留的内容：
- 所有人名（首次出现标注英文原名 + 一句话身份）
- 所有书名、论文名、概念名（首次出现标注英文）
- 所有具体数据、年份、金额、地点
- **所有案例的完整细节**（谁、什么、什么时候、在哪里、结果如何）
- 讲师的引用来源（"这是 Karen Hao 书里写的..."）
- **访谈类视频：双方的对话互动、质疑、反问、补充，都要保留**

允许合并的内容：
- 纯寒暄（"大家好"）→ 融入开头一句带过
- 技术操作（"我们看下一段"）→ 融入过渡句
- 重复强调 → 合并为一次表达，注明"讲师多次强调"

### 3. 保留修辞风格

- 讽刺/反讽：直接引用，注明"讲师讽刺地说..."
- 夸张/戏剧化：保留原文，注明"讲师用夸张的语气说..."
- 反问：保留反问结构，不改成陈述句
- 关键强调：用 **加粗** 标出
- 引经据典：注明"讲师引用了..."

### 4. 术语处理

- 正文用中文叙述
- 专业术语、人名、书名首次出现时加括号保留英文 + 身份/背景
- 后续直接用中文简称
- 禁止在文末单独做"概念表"——术语在正文里解释，概念速查表是复习用的不是定义用的

## 字数与密度要求（硬性指标）

### 总字数下限（按视频时长，仅统计主体内容）

| 视频时长 | 笔记最低总字数 | 说明 |
|----------|---------------|------|
| < 15 分钟 | ≥ 2,500 字 | 短视频也要有深度 |
| 15-30 分钟 | ≥ 4,500 字 | |
| 30-60 分钟 | ≥ 8,000 字 | 1小时约相当于8-10页A4 |
| 60-120 分钟 | ≥ 12,000 字 | |
| > 120 分钟 | ≥ 16,000 字 | 长篇采访必须有分量 |

> 访谈类和授课类按上表执行。教程类可适当下调（操作演示占时间但不占文字），但核心讲解部分必须达标。
>
> **字数达标后停止，不要过度展开。超20%为上限。** 如果已经充分还原了推导过程，不要为凑字数而添加冗余过渡、重复解释或空洞铺陈。

### 每个子论点的字数按推导步骤计算

- 简单论证（1-2步）：不少于200字
- 中等论证（3-4步）：不少于300字
- 复杂论证（5步以上）：不少于400字
- 每个论证步骤不少于30字

> **字数是下限，不是目标。达标即可，不要硬凑。** 一个步骤如果20字就能说清楚逻辑，不要硬扩到50字。信息密度 > 字数长度。
>
> **避免口语化逐句还原**：不要写"他说…然后他说…接着他又说…"这种翻译腔。提取逻辑骨架，用因果关系连接，而不是按时间顺序复述每一句话。

### 案例完整性要求

每个案例/故事在笔记中的字数不得少于它在原字幕中所占字数的 **50%**。
> 如果一个 3 分钟的精彩故事（约 450 字），你只用 30 字带过 → 立即返工。

### "总结感"自测 + 过程完整性检查

写完每个主题后检查，出现以下情况必须重写：
- 某个子论点只有论点句、没有展开
- 读起来像"这本书主要讲了…然后又讲了…最后讲了…"
- 用"本质上""归根结底""简而言之""总体而言"等总结词开头
- 全文没有任何具体的时间、金额、数字、人名细节
- 每个段落感觉像"把 10 分钟压缩成了 2 行"

**新增——过程完整性检查（最重要）**：
- 随机挑3个子论点，问自己：讲师在这个话题上用了几步推导？笔记保留了相同数量的步骤吗？
- 如果讲师用了5步推导，笔记只写了1步（"总之他认为..."）→ **返工**
- 如果子论点开头50字以内就出现了结论性表述（"X认为Y是Z""X的核心观点是""X证明了"）→ **返工**，把结论移到段落末尾
- 如果读完子论点后，感觉"这段话告诉我'结论是什么'，但没告诉我'怎么推导的'"→ **返工**

## 十三条铁律（绝对禁止）

1. ❌ **遗漏**核心信息：人名、书名、案例、数据，缺一项都不行
2. ❌ **机械罗列**：连续使用"讲师+动词"句式超过 2 句
3. ❌ **总结式写作**：把 10 分钟内容压成 2 行，没有展开和细节
4. ❌ **压平语气**：把反问改陈述、把讽刺改中性、把戏剧化改平淡
5. ❌ **空洞套话**："这个问题非常复杂""值得深入探讨"等废话
6. ❌ **附录式结构**：文末放人物索引、概念表、引语集（概念速查表除外——它是学习工具不是附录）
7. ❌ **到处时间戳**：仅主题开头标注一次时间段，正文里不加
8. ❌ **模糊表述**：写"等等""之类""一些""大致"——不确定就去查，查不到就说明
9. ❌ **结论先行**：不要在子论点开头就给出结论（"X认为...""X的核心观点是...""X证明了..."）。结论必须在推导过程末尾自然浮现。如果子论点开头50字以内出现结论性表述，立即重写
10. ❌ **一步到位压缩推导**：如果讲师用了3步推导到达结论，笔记必须保留3步，不能压缩成1步（"总之他认为..."）
11. ❌ **只记最终回应**：如果讲师先提出质疑/反对意见再逐步回应，必须保留质疑→回应的完整来回，不能只写最终回应结论
12. ❌ **只记嘉宾最终答案**：访谈中嘉宾先犹豫/部分回答/补充/修正，必须保留完整过程，不能只保留最终答案
13. ❌ **口语化逐句还原**：禁止写"他说…然后他说…接着他又说…"这种翻译腔。提取逻辑骨架，用因果关系连接，不是按时间顺序复述每一句话。如果一段推导可以用3句话说完逻辑，不要硬扩到10句话
